# 制度审查智能体系统

## 项目概述

### 项目背景

制度审查智能体是一个基于大语言模型（LLM）构建的企业制度全生命周期智能化审查工具。该系统旨在帮助企业实现制度条款审查的自动化、标准化处理，提供全面、精准、高效的制度审查服务。

### 核心功能

1. **智能问答**：基于知识库的检索增强生成（RAG），支持上下文感知的智能对话
2. **文件解析**：支持 PDF、Word、TXT 等多种格式文件的解析和内容提取
3. **知识库管理**：支持创建、管理和检索知识库，实现文档向量化存储
4. **文件操作**：支持文件标注、修改、审查报告生成等操作
5. **工具调用**：智能体可自主调用工具完成文件修改和标注任务
6. **规则管理**：支持全局规则和对话规则的创建、管理和应用

### 核心价值

- **合规性审查**：校验制度条款是否与现行法律法规相抵触
- **一致性审查**：校验条款与上位制度、内部现有制度是否一致
- **适用性审查**：评估条款是否符合企业管控流程要求
- **形式审查**：排查错别字、格式混乱、逻辑不清晰等问题

## 核心模块说明

### 1. 智能体模块 (agent/)

#### audit_agent.py

基于 LangGraph 实现的智能体核心，包含：

- **状态管理**：使用 `AgentState` 管理对话状态
- **工作流图**：构建检索→智能体→工具的工作流
- **工具调用**：支持自主调用文件操作工具
- **迭代控制**：防止无限循环
- **自定义工具节点**：`ConversationAwareToolNode` 解决 ContextVar 传递问题

```python
class AgentState(TypedDict):
    messages: Annotated[Sequence[BaseMessage], add_messages]
    question: str
    context: Optional[str]
    rules_context: Optional[str]
    knowledge_base_id: Optional[int]
    conversation_id: Optional[int]
    file_content: Optional[str]
    file_paths: Optional[List[str]]
    iteration_count: int

class AuditAgent:
    def __init__(self, enable_tools: bool = True)
    def _init_llm(self)           # 初始化大语言模型
    def _build_graph(self)        # 构建工作流图
    def chat_with_tools(self)     # 带工具调用的对话

class ConversationAwareToolNode:
    """自定义工具节点，确保在工具执行时 conversation_id 可用"""
    def __init__(self, tools: list)
    def __call__(self, state: dict, config: RunnableConfig = None) -> dict
```

## 规则系统

### 规则类型

| 类型 | 说明 | 作用范围 |
|------|------|----------|
| `global` | 全局规则 | 对所有对话生效 |
| `conversation` | 对话规则 | 仅对特定对话生效 |

### 规则优先级

```
对话规则 > 全局规则 > 向量库检索内容
```

### 规则应用流程

```
1. 用户添加规则（通过对话或API）
2. 规则存储到数据库
3. 对话时，_retrieve_node 检索规则
4. 规则格式化为 rules_context
5. rules_context 注入到系统提示词
6. LLM 根据规则生成响应
```
